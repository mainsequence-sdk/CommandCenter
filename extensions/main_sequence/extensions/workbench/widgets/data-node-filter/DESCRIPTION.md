Stores and publishes a reusable Main Sequence DataNode dataset for workspace composition. Use this as the standard, canonical widget when a workspace needs to request tabular data from the Main Sequence Data Platform, reuse that dataset across several downstream widgets, or feed Data Node Table, Data Node Graph, Statistic, Curve Plot, or Zero Curve widgets from one shared source.

The widget can query a direct Main Sequence DataNode, consume another bound Data Node, or publish a manually authored table. Its optional inbound port is `sourceData`, which accepts `core.tabular_frame@v1`; when bound, the upstream source controls the effective data selection and preview rows. Its outbound port is `dataset`, which publishes the canonical `core.tabular_frame@v1` payload with `columns`, `rows`, optional `fields`, source metadata, status, range, and error information for downstream widgets.

Prefer this widget when the DataNode should remain the reusable source of truth and the required output shape can be produced with the widget's built-in simple transformations. `none` publishes the input dataset as-is, except for optional final projection. `aggregate` groups by selected key fields and reduces each group with `first`, `last`, `sum`, `mean`, `min`, or `max`. `pivot` uses key fields as row dimensions, turns values from one categorical field into output columns, and fills those columns from a selected value field using the aggregate mode. `unpivot` melts selected wide value columns into long-form rows, preserving selected key fields and writing output field names such as `series` and `value`. `project columns` is an optional final step that keeps only selected published columns after the main transform.

The transform order is always: input dataset -> one transform mode -> projection -> published dataset. A single Data Node does not chain multiple transform modes; when a workspace needs multiple reshapes, chain multiple Data Node widgets so each node publishes one canonical dataset to the next hop.
